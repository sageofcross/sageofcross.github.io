距離上次碰部落格也快六年了，突然開始接觸當然有點原因，讓我依序來講吧。

## 為何荒廢
誠實點，最初弄的目標就是想豐富履歷，在面試的時候可以說出：「個人有在寫技術文章與分享新知，可以參考我的部落格」。並非我真的對撰寫文章非常有熱情，或是非常希望分享學到的知識等。

話雖如此，實際動手利用 [Hexo](https://hexo.io/zh-tw/) 與 [GitHub Page](https://docs.github.com/en/pages) 搭建時還是很開心，不禁想著如果持續寫下去是不是有機會成為什麼技術大神。最終因怠惰與工作繁忙沒有堅持下去，逐漸地荒廢。

## 嘗試重來
荒廢後不止一次想過重啟，但每次都敗在自己無意義的堅持。

使用 Hexo 時我用了一個很合口味的主題 [Cactus](https://github.com/probberechts/hexo-theme-cactus) ，即使到現在我依然覺得這個主題超棒，但未曾想過 Cactus 變成了我對部落格的最低要求。

對程式設計師來說，有機會從頭來過時絕對會想嘗試新玩具，當時跑去研究 [Hugo](https://gohugo.io/) 怎麼用，腦中想的有：

- Node.js 好大一包
- Hexo 當文章多的時候渲染可能要到秒級
- Hugo 使用 Go 語言，我比較熟悉
- Hugo 當時開始紅

然而還記得前面提到的嗎？對， Hugo 並沒有 Cactus 主題，於是天真的我就跳入了搬遷主題的坑。

那個只記得基礎 HTML 、 JS 、 CSS 三元素的我，實際下去看 Cactus 時看到的是 SCSS 、 Stylus 、 EJS 之類的全新玩意，原本對前端就不太熟了，現在有更多需要學的東西瞬間讓我難以消化，更別提不同的 SSG 在處理邏輯、 Meta 傳遞等地方也不盡相同，導致想要完整復刻出 Cactus 的我不斷受挫，最後只能將半成品封印在雲端中。

後來一段時間後看到 [Zola](https://www.getzola.org/) 時，不知道是時間抹滅了挫敗感還是覺得自己變強了，再次跳入相同的坑並跌的同樣慘。

## 下定決心
首先 2025 年底時被資遣，因為公司原本的新產品決定不做，所以相關人員無一例外的說再見。不過我當時是樂觀的，還想著剛好 FF14 台版要上了，當作臨時撿到的長假好好放鬆，等過年之後再重新開始找。

實際 2026 年初時真的有一種全新開始的感覺：

1. 過年前有間不錯的公司面試有推進，且看來蠻有希望的
2. 房東替租屋處換變頻冷氣，處理壁癌等雜事
3. 簽了中華有線網路並弄好 PC

但隨著過完年，彷彿像是要懲罰那個過於樂觀的我似的，新開始的感覺突然就變成了多個打擊：

1. 那間不錯的公司回覆了感謝信
2. 變頻冷氣零件斷裂，租屋處開始出現老鼠（七、八年來第一次）
3. 中華網路有時會莫名的延遲

當然這些都不算什麼大事，但同時間一起亂掉的巧合讓我很難不去多想到底是怎麼了？

因此這段時間我沒有馬上開始找工作，總覺得有些東西需要先想清楚，又想到了公司感謝信中提到 AI ，驚覺自己對於 AI 在這個大時代的影響力似乎過於輕視了，雖然平時開發與學習會使用到 AI ，但心中一直覺得：「 AI 距離要取代程式設計師還遙遠的」、「用 AI 寫出來的東西使用上肯定會有很多問題」。

現實是 AI 發展速度拿日新月異比喻都嫌不夠，用秒新分異可能還恰當點，想到這不禁覺得自己很可笑。

未來幾乎不用思考就能回答出是 AI 的世界，身為受害者之一的程式設計師該做的才不是抱著那種毫無根據的自信得過且過，應該要開始且比任何人都認真的學習好這個新時代的工具才對。只有這樣才能提升自己的價值。

於是部落格就是我用來學習與 AI 合作的成品，但我還是先以傳統的對話形式協作，而非近期較紅的 Vibe Coding 或 Agent 等。

## 我的感受
本次選擇 Claude 來輔助幫忙，但因為用的是傳統的我貼上問題請他解決，所以感想很單純。

Claude 對於程式碼的理解能力有點超越我當初使用很多，大多數的問題都能快速且簡單地提出解決辦法。這對於前端菜鳥的我來說可以很輕易的修改不懂的程式碼，且如果想要學習 Claude 也能給出很詳盡解釋與資料來源，不用像以前還會擔心回答是否為幻覺。

當然過程中也還是有碰到問題：

- Claude Web UI 似乎渲染回應中有 HTML Tag 時會吃掉，例如有次回答包含 `<a>` 在我這邊完全看不到，但 Claude 可以所以他一直不知道錯在哪邊
- 比較複雜的問題只靠描述會需要來回多次，且 Claude 很高機率會自己走歪
- 解法似乎會假定某個方向，只有當我提出或許可以怎麼做的時候才會嘗試往那走

這次協作我感覺到 Claude 確實能夠取代初階的程式設計師，無論是回應速度還是實作能力都樂勝。

但也很明顯的離資深還有一段距離，只是這段距離已經不如我當初想得那麼遠而已，之後嘗試 Vibe Coding 或 Agent 時不知道會不會再次被驚訝到。

最後聊個突然的感受，這個時代 AI 對於學習來說感覺會是把雙面刃，因為知識學習更簡單難度更低，以前用關鍵字搜尋後自身還要有一定能力判斷資料能不能用，現在 AI 整條路都幫你鋪好時，那種感覺真的是相當舒服，對於真的要學習的人來說想必可以提升好幾倍的速度吧。

也因為學習變得超級簡單，對於垂手可得的東西人往往都難以珍惜，最擔心的就是會使人抱持著「反正我問 AI 就知道了」的想法而不再深究吧，反正出了事情有 AI 一切都會沒事的。

這也讓我目前對於能生在有 AI 的時代感到既開心又難過，真的是很複雜的心情。
